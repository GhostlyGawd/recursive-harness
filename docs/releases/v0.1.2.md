# Recursive Harness v0.1.2

`v0.1.2` is the first governed distribution release of Recursive Harness. It turns the
existing agent-development system into a versioned, checksummed, documented active beta
without hiding the boundaries that remain outside deterministic CI.

## Highlights

- Complete evidence loop: predictions, outcomes, calibration, corrections, follow-ups,
  retrospectives, GC, Scorecard, feature flags, and privacy controls.
- Concurrent work safety: worktree/session ownership, trunk leases, branch-first and
  pre-merge guards, explicit human approval, and immutable companion-repository revisions.
- Structural intelligence: Cartograph, Atlas, Pulse, health/rot checks, and structural Q&A.
- Governed coordination: proposal lifecycle enforcement, Fleet/Agent Mail, Mission Control,
  account silos, project integration, and Bash/PowerShell launchers.
- Profile-aligned Signal Loop identity plus real Doctor, Scorecard, calibration, Cartograph,
  and fixture-backed Mission Control evidence on the README landing page.
- Receipt-bound Recursive Observe, Recursive Learn, Recursive Verify, and local-only
  Recursive Coordinate packages for generic Agent Skills, Claude Code, and Codex.
- A separate Recursive Lab generated-experimental package for preview-only Brainstorm and
  Roadmap workflows, plus separately trusted Guard and narrow Specialization adapters.

## Distribution and compatibility

- Repository-wide MIT License.
- Deterministic `tar.gz` and ZIP archives built from the tagged commit.
- Embedded `RELEASE-MANIFEST.json` with per-file SHA-256 values plus one published checksum
  sidecar covering both archives.
- Non-destructive Bash and PowerShell uninstall paths that preserve settings, transcripts,
  backups, ignored state, and the checkout.
- Supported baseline: Python 3.12, Git 2.39.0+, Bash 5.x/current Git Bash, current hosted
  Ubuntu/Windows/macOS, and Claude Code 2.1.200+.

Download both archives and the shared checksum sidecar, then verify before installation:

```bash
gh release download v0.1.2 -R GhostlyGawd/recursive-harness \
  -p "recursive-harness-v0.1.2.tar.gz" \
  -p "recursive-harness-v0.1.2.zip" \
  -p "recursive-harness-v0.1.2.sha256"
sha256sum -c recursive-harness-v0.1.2.sha256
```

On macOS use `shasum -a 256 -c`; on Windows compare each archive's
`Get-FileHash -Algorithm SHA256` result with its sidecar line.

## Breaking changes

No intentional breaking change is declared from the `v0.1.0` public snapshot. The project
is still an active beta, so operators should use the documented settings refresh and Doctor
checks instead of assuming internal state schemas are stable.

## Security and privacy

- Secret scanning, push protection, private vulnerability reporting, Dependabot, CodeQL,
  immutable Actions pins, owner-only local-state handling, redaction, retention, audit, and
  scrub controls are enabled.
- The release review removed weak hashing, unsafe session-derived filenames, eval traversal,
  mutable distributed revisions, regex backtracking paths, and a duplicate staging tree.
- Protected-main Python and Actions CodeQL pass with zero open CodeQL alerts. Secret scanning
  and Dependabot also report zero open alerts at the release gate. Scanner counts are evidence
  from one point in time, not a sandbox or a guarantee that defects do not exist.

## Upgrade

Stop active sessions, preserve `.claude-private/` and `state/`, fast-forward or extract the
new version, rerun `./install.sh`, refresh account settings with
`./account-init.sh --all --sync-settings`, and run `python3 bin/harness doctor`. See
[Compatibility and upgrades](../compatibility.md) for rollback and removal.

## Known limitations

- Active beta: breaking operator-visible changes can occur before 1.0.
- Trusted local automation, not a sandbox; hooks and commands run with operator permissions.
- Deterministic CI does not invoke a model API. Real Claude Code replay remains human release
  acceptance evidence.
- Mission Control and Fleet MCP are optional dependency-bearing surfaces.
- GitHub Releases are the packaged distribution channel; native package-manager channels are
  not published yet.
- Public marketplace discovery, hosted-web execution, automatic model skill selection, and
  multi-machine Coordinate consensus remain unverified or unsupported.
