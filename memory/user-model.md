## Working style

- when adapting another repo's artifact, COPY it into the target repo and edit the copy — never modify the source repo (evidence: 1, last: 2026-06-14, source: stated)
- iterate harness/skill changes in an isolated git worktree and test end-to-end before merging or replacing anything — and "end-to-end" means exercising the REAL path/trigger (the actual tool, not a proxy or just green gates on artifacts); a self-flagged-unverified load-bearing claim is a ship-blocker, not a footnote (2026-06-20: merged a "rides into worktrees" change while its own prediction 55b1735b flagged that behavior unverified — post-merge it was false, forcing a full re-delivery).
