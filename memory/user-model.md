## Working style

- no destructive or irreversible action until it is verified end-to-end AND explicitly approved; prefer non-destructive paths (copy-not-move, new branch/worktree, additive/reversible edits) and minimize blast radius (evidence: 1, last: 2026-06-14, source: stated)

The test: has the user already approved this batch/scope or settled this point ("do all of X", "merge it", "yes"/"do it", "you're approved on everything")? If YES and the next step is a PROCESS action — merging, deploying, starting the next item, cascading — DO IT, do not ask. Merging INCLUDES enforcement-layer PRs (hooks/lint/evals/settings): prime directive 5's "a human merges" is SATISFIED when the human DIRECTS the merge, so running `gh pr merge` under that instruction is human-in-the-loop, NOT a unilateral self-merge — do not hand the merge button back when told to land it (2026-06-27: I said "over to you for the keystone merge"; user: "No you merge"). STOP only for (a) a genuine NEW risk, or (b) an unsettled taste/policy/ambiguous-intent FORK the user must decide (still confirm THAT, per the planning/align bullet). FLOORS "everything" never overrides: guard-WEAKENING and destructive/irreversible ops — refuse or flag even when told "do everything" (prime directive 5 + the no-destructive-action floor).
